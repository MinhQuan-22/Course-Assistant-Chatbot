# Điều này giúp bạn vẫn có thể sử dụng file này bình thường với lệnh:
#       pip install -r requirements.txt
# mà không gặp lỗi syntax.
#
# ==========================================
# 1. YÊU CẦU HỆ THỐNG (PREREQUISITES)
# ==========================================
# - Node.js: v18+ (Dành cho việc chạy Frontend - Vite/React)
# - Python: 3.9+ (Dành cho Backend - Django)
# - MySQL Server: 8.0+ (Hoặc chỉ cần Docker để chạy qua file docker-compose.yml)
# - Docker & Docker Compose (Dành cho việc chạy MySQL & ChromaDB nhanh chóng)
#
# ==========================================
# 2. CẤU HÌNH BIẾN MÔI TRƯỜNG (.env)
# ==========================================
# Yêu cầu: Tạo một file tên là `.env` đặt tại thư mục gốc (root) của dự án.
# Nội dung của file .env cần bao gồm các thông số sau:
#
# [Django Config]
# DJANGO_SECRET_KEY=change_me_to_a_secure_key
# DJANGO_DEBUG=1
# DJANGO_ALLOWED_HOSTS=localhost,127.0.0.1,backend
#
# [Database Config]
# # Cấu hình mặc định trong source code đang dùng MySQL ở port 3307
# MYSQL_DATABASE=course_assistant_db
# MYSQL_USER=root
# MYSQL_PASSWORD=root
# MYSQL_HOST=127.0.0.1
# MYSQL_PORT=3307
#
# # Nếu bạn sửa để xài Postgres thì cấu hình các biến dưới:
# POSTGRES_DB=course_assistant
# POSTGRES_USER=postgres
# POSTGRES_PASSWORD=postgres
# POSTGRES_HOST=postgres
# POSTGRES_PORT=5432
#
# [Cấu hình gửi Email (SMTP)]
# EMAIL_HOST_USER=youremail@gmail.com
# EMAIL_HOST_PASSWORD=your_app_password
#
# [Vector DB & AI Models]
# CHROMA_HOST=127.0.0.1    # Hoặc 'chroma' nếu chạy mạng docker 
# CHROMA_PORT=8001         # Port map ra ngoài từ docker-compose (8001:8000)
# OPENAI_API_KEY=your_openai_api_key_here
# GEMINI_API_KEY=your_gemini_api_key_here
#
# ==========================================
# 3. HƯỚNG DẪN KHỞI TẠO DATABASE & VECTOR DB (Bằng Docker)
# ==========================================
# 1. Mở terminal và di chuyển vào thư mục backend:
#    cd backend
# 2. Chạy docker-compose để khởi tạo MySQL và ChromaDB:
#    docker-compose up -d
# 3. Docker sẽ tự động chạy script `schema.sql` và `seed.sql` để tạo sẵn
#    dữ liệu (seed data) trong MySQL trong lần đầu khởi chạy.
#
# ==========================================
# 4. HƯỚNG DẪN CÀI ĐẶT & CHẠY BACKEND (Django Server)
# ==========================================
# 1. Di chuyển vào thư mục backend/:
#    cd backend
# 2. Khởi tạo môi trường ảo (Virtual Environment):
#    python -m venv venv
# 3. Kích hoạt môi trường:
#    (Mac/Linux): source venv/bin/activate
#    (Windows): .\venv\Scripts\activate
# 4. Cài đặt các thư viện Python (dùng luôn file này):
#    pip install -r ../requirements.txt 
# 5. (Tùy chọn) Chạy migrate của Django nếu có chỉnh sửa model:
#    python manage.py makemigrations
#    python manage.py migrate
# 6. Khởi chạy server:
#    python manage.py runserver
#    -> Backend chạy tại: http://localhost:8000
#
# ==========================================
# 5. HƯỚNG DẪN CÀI ĐẶT & CHẠY FRONTEND (React/Vite)
# ==========================================
# 1. Mở một terminal mới, di chuyển vào thư mục frontend/:
#    cd frontend
# 2. Cài đặt các thư viện node_modules:
#    npm install
#    (Lưu ý: Các dependencies chính bao gồm React 18, TailwindCSS, 
#            shadcn-ui, react-router-dom, react-query, axios, v.v. 
#            đã được cấu hình tự động trọn gói trong package.json).
# 3. Khởi chạy ứng dụng web:
#    npm run dev
#    -> Frontend chạy tại: http://localhost:8080 (hoặc http://localhost:5173)
#
# ==========================================
# 6. FILE CHỨNG NHẬN (CERTIFICATES) / KEYS KHÁC
# ==========================================
# - GOOGLE_CLIENT_ID (Dành cho Google Login) đã được nhúng sẵn vào settings.py
# - Vui lòng không commit file .env (đã được đưa vào .gitignore)
